# 痔疮科普网站设计方案

## 项目概述
为办公族、程序员等久坐人群开发的痔疮科普网站，提供科普知识、医院/科室排名、用户评论功能。

## 技术栈
- **前端框架：** Preact (轻量级React，10KB)
- **构建工具：** Vite
- **样式方案：** Tailwind CSS
- **后端服务：** Supabase (PostgreSQL + 实时订阅)
- **部署：** 静态托管 (Vercel/Netlify/Cloudflare Pages)

## 页面结构

### 1. 首页 (/)
- Hero区域：轻松幽默的标语（如"程序员的第一道坎"）
- 快速入口：科普知识、找医院、症状自测
- 热门医院推荐（评分最高的3-5家）
- 最新科普文章（3篇）

### 2. 科普知识列表 (/knowledge)
- 分类筛选：痔疮类型、预防方法、治疗方式、术后护理
- 文章卡片：标题、摘要、阅读时间、阅读量
- 分页或无限滚动

### 3. 科普文章详情 (/knowledge/:slug)
- Markdown渲染文章内容
- 文章目录（TOC）
- 相关文章推荐
- 分享功能

### 4. 医院/科室列表 (/hospitals)
- 搜索栏：支持城市、医院名搜索
- 筛选器：城市、医院等级、科室类型
- 排序：评分、评论数、距离
- 医院卡片：名称、等级、评分、科室标签、地址

### 5. 医院详情 (/hospitals/:id)
- 医院基本信息
- 科室列表
- 评分统计（平均分、各星级占比）
- 用户评论区（分页）
- 写评论表单（匿名）

### 6. 提交医院 (/submit)
- 医院信息表单（名称、城市、地址、等级）
- 科室信息
- 提交成功提示（待审核）

## 数据库设计 (Supabase)

### hospitals 表
- id, name(医院名), city(城市), address, level(三甲/三乙等), departments(科室JSON), rating(平均分), review_count, status(审核状态), created_at

### reviews 表
- id, hospital_id, rating(1-5星), content, nickname, ip_address, created_at

### knowledge_articles 表
- id, title, slug, category, content(markdown), views, created_at

## 核心功能

### 1. 搜索功能
- 前端实现：fuse.js（轻量级模糊搜索）
- 支持：医院名、城市、症状关键词
- 实时搜索建议

### 2. 评论系统
- 匿名评论，无需登录
- 防刷机制：IP限制、内容长度限制、敏感词过滤
- 实时显示新评论

### 3. 评分统计
- 实时计算平均分
- 星级分布图表
- 按评分排序

## 视觉设计

### 风格：轻松友好
- 主色调：温暖的橙色/珊瑚色（#FF6B6B）+ 深蓝灰（#2D3748）
- 圆角设计（16px border-radius）
- 柔和的阴影效果
- 适合程序员的代码块样式

### 响应式断点
- 移动端：< 640px
- 平板：640px - 1024px
- 桌面：> 1024px

## 性能优化
- 图片懒加载
- 路由懒加载
- 骨架屏加载状态
- CDN加速（静态资源）

## 初始内容

### 科普文章主题
1. **痔疮类型**：内痔、外痔、混合痔的区别
2. **久坐危害**：为什么程序员容易得痔疮
3. **预防措施**：办公室运动、饮食建议
4. **治疗方法**：药物、手术、微创对比
5. **术后护理**：恢复期的注意事项

### 示例医院数据
- 北京：协和医院、中日友好医院
- 上海：长海医院、瑞金医院
- 广州：中山大学附属第六医院
- 成都：华西医院
